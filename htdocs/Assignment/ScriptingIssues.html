SQL Injection

This is where someone injects or changes SQL codes to access data they are not permitted to see or change. This can be very dangerous as an attacker would be able to access the database containing user information, and if not properly secured or encoded they can access personal details like bank accounts and card details. They can also delete all this information, which would then put the website owner at fault under the Data Protection Act for not keeping it secure.

Cross-Site Scripting

This where an attacker would execute code to add malicious scripts into a website. This would then execute the script onto any clients browser whenever they loaded the website, which in turn could install viruses or other harmful software that the attacker could use to gin access to a client’s PC or data.

Bad Scripting Leaving Backdoors

Backdoors are implemented by coders that only they know of if they ever need to bypass or access anything on their website with administrative intent such as troubleshooting. However, this is very risky as if not done right and leaving this back door easily accessible attacks may be able to access it and perform malicious intent such as taking user information or installing software that downloads harmful software to client PCs.

SSL Hijacking

This is where a hacker gains access to a client’s session on a web server without the client being aware of this. This can be used to see user inputs and access their account later to collect their information, change or delete it. This is essentially making your identity on the website the client’s and taking over what they do and making it look like they did it.

Database Liabilities

This is when a website’s code isn’t well developed leaving holes in it and ways for the database and user information to be accessed. If code is not debugged and checked and tested thoroughly it can lead to many errors in the websites functionality and can lead to attackers exploiting this error to their own malicious intentions such as accessing a database. This can also be done by tricking the user input system if it’s not validated and gaining access to the database of users from there.

Faking Statistics

This is where a website owner would use software to change and falsify their analytic statistics of how many views and clicked they receive on their website. This is done to gain recognition from advertising companies and receive top brand ads on a website, increasing the revenue you receive from the website and the clicks on these ads.
